# AI 小说精读助手

基于大语言模型的浏览器端智能小说阅读工具。上传 TXT/EPUB 文件，配置任意大模型 API Key，即可对小说进行章节总结、人物关系分析、剧情时间线梳理、AI 问答等深度交互。

**所有数据存储在浏览器本地，API 调用直接从浏览器发出，不经过任何第三方服务器。**

## 功能

### 小说导入
- 支持 TXT、EPUB 格式，自动检测文件编码（GBK / Big5 / UTF-8）
- 智能识别中文章节标题（"第一章"、"Chapter 1"、数字编号等模式）
- 文件夹批量导入，书架式管理，显示字数、章节数、阅读进度

### AI 章节分析
- 一键生成当前章节摘要（核心情节、关键人物、伏笔、主题推进）
- 批量总结全书所有章节
- 范围总结：自定义章节区间（如第 5-12 章）的临时分析

### 全书分析
- **剧情时间线** — 提取 15-25 个关键事件，标注类型和因果关系
- **人物关系图谱** — 自动识别角色，生成可拖拽的力导向关系图，支持全屏放大
- **全书总览** — 故事主线、主题分析、结构特点、阅读建议

### AI 问答
- 支持多轮对话，追问小说内容
- 对话历史保留，可随时开启新会话
- 所有对话为临时会话，不保存到本地

### 笔记
- **章节笔记** — 为每个章节独立记录笔记，切换章节自动切换笔记
- **全书笔记** — 跨章节的全局笔记
- **AI 收藏** — 将 AI 回答一键收藏到笔记（支持章节/全书两种分类）
- **手风琴展开** — 笔记默认折叠显示前两行，点击展开全文，同一时间只展开一条
- **章节移入全书** — 章节笔记可随时移入全书笔记
- 所有笔记存储在浏览器本地 IndexedDB，可单独删除

### 阅读体验
- 字体大小和粗细自由调节
- 暗色 / 亮色模式一键切换
- 左侧章节目录可展开收起，底部固定导航栏支持上下章切换
- 键盘快捷键：← → 切换章节
- 阅读进度自动记忆，下次打开自动恢复
- 移动端响应式适配

## 快速开始

**前置条件：** 安装 [Node.js](https://nodejs.org)（v18 或以上）。

```bash
# 克隆仓库
git clone https://github.com/AscendJK/ai-novel-reader.git
cd ai-novel-reader

# 安装依赖（仅首次需要）
npm install

# 启动开发服务器
npm run dev
```

打开浏览器访问 `http://localhost:5173`。

### Windows 快捷脚本

| 文件 | 功能 | 用法 |
|------|------|------|
| `start.bat` | 一键启动 | 双击，选择模式 1（开发/热更新）或模式 2（生产/手机稳定访问）。首次自动安装依赖。 |
| `stop.bat` | 停止服务 | 双击，自动关闭端口 5173 的服务器进程。 |
| `port-mgr.bat` | 端口管理 | 双击打开管理菜单：查看运行中的端口、启动服务、关闭指定端口或全部端口。 |

`start.bat` 两种模式：
- **模式 1（Dev）**：开发模式，代码修改即时热更新，但手机切后台回来可能刷新
- **模式 2（Prod，推荐手机用）**：先构建再启动，无热更新，手机切后台不会刷新

## 使用指南

### 配置 API

1. 点击右上角设置图标
2. 选择 API 提供商（OpenAI / Anthropic / DeepSeek / 自定义兼容接口）
3. 填写 API Key、选择模型
4. 点击保存 — API Key 仅存储在浏览器本地

### 阅读与分析

1. 在书架页面上传小说文件
2. 点击卡片进入阅读视图
3. 点击右上角「AI 分析」打开分析面板
4. 选择「本章分析」生成章节摘要
5. 切换到「全书分析」标签页生成深度分析
6. 使用「问答」标签页进行自由提问

### 局域网共享

```bash
npm run build
npx vite preview --host 0.0.0.0 --port 5173
```

同一 WiFi 下的手机或平板访问 `http://你的电脑IP:5173` 即可使用。不同设备之间的数据完全隔离。

## 技术架构

```
┌──────────────────────────────────────────────┐
│  React 19 + TypeScript + Vite                │
├──────────────────────────────────────────────┤
│  多 Agent 协作引擎                             │
│  ├ 分块 Agent    — 语义切割，构建索引           │
│  ├ 总结 Agent    — 章节/全书摘要               │
│  ├ 人物 Agent    — 角色识别，关系图谱 JSON      │
│  ├ 时间线 Agent  — 事件提取，因果链             │
│  ├ 图谱 Agent    — 独立生成关系图 JSON          │
│  └ 协调 Agent    — 任务调度，上下文管理          │
├──────────────────────────────────────────────┤
│  API 调用层                                    │
│  ├ OpenAI / Anthropic / DeepSeek / 兼容接口    │
│  ├ Token 预算管理 + AbortController            │
│  └ 错误分类（auth/network/context_length）     │
├──────────────────────────────────────────────┤
│  存储层                                        │
│  ├ IndexedDB (Dexie.js) — 小说、分析、设置      │
│  └ localStorage — 主题、字体、阅读进度          │
└──────────────────────────────────────────────┘
```

**核心技术：**
- **React 19 + TypeScript + Vite** — 前端框架
- **Tailwind CSS + shadcn/ui** — UI 组件
- **Zustand** — 状态管理
- **Dexie.js** — IndexedDB 封装
- **d3-force** — 人物关系力导向图谱
- **react-markdown** — Markdown 渲染
- **JSZip** — EPUB 文件解析
- **TF-IDF / BGE Small ZH** — 本地语义检索引擎
- **Transformers.js** — 浏览器端 ONNX 模型推理

## 本地检索引擎

项目内置了 **BGE Small ZH v1.5** 中文嵌入模型（约 26MB，Xenova INT8 量化版），支持将小说全文在浏览器本地向量化检索。当你使用全书分析或 AI 问答时，系统会自动检索与问题最相关的段落发给大模型，而不是随机截取章节样本。

可选的检索引擎：

| 引擎 | 大小 | 说明 |
|------|------|------|
| TF-IDF（内置） | 0 MB | 纯字符级检索，即时可用 |
| BGE Small ZH（内置） | ~26 MB | 中文专精，语义级匹配 |
| 自定义模型 | 任意 | 支持 Xenova 转换版模型 |

详见设置页的「本地检索引擎」面板。

## 数据隐私

| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| 小说原文 | 浏览器 IndexedDB | 不上传任何服务器 |
| AI 分析结果 | 浏览器 IndexedDB | 按小说归类，可单独删除 |
| API Key | 浏览器 IndexedDB | 仅在调用 API 时使用 |
| UI 偏好 | localStorage | 主题、字体、阅读进度 |
| AI 问答对话 | 仅内存 | 刷新页面后消失 |

- 删除小说时自动清除所有关联数据（原文、分析、图谱、进度）
- 不同端口的访问数据完全隔离（localhost:5173 ≠ localhost:5174）
- Vite 开发服务器仅发送静态文件，不存储或中转任何用户数据
