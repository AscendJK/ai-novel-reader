# AI 小说精读助手

浏览器端 AI 小说阅读工具。上传 TXT/EPUB，配置任意大模型 API Key，即可获得章节总结、人物关系、剧情时间线、AI 问答等深度阅读辅助。

支持多设备同步——所有数据存储在服务端 SQLite，一个用户名即可在电脑、手机间无缝切换。

## 快速开始

**前置条件：** [Node.js](https://nodejs.org) v18+

```bash
git clone https://github.com/AscendJK/ai-novel-reader.git
cd ai-novel-reader
npm install
```

### 启动

```bash
./start.sh    # Linux/macOS，选 1=开发 2=生产
start.bat     # Windows 双击，选 1=开发 2=生产
```

打开 `http://localhost:5173`。

| 模式 | 说明 |
|------|------|
| 开发（1） | 代码热更新，但手机切后台可能刷新 |
| 生产（2） | 先构建再启动，手机访问稳定 |

停止：`./stop.sh` 或 `stop.bat`

## 使用教程

### 1. 登录

首次访问弹出登录框：
- **创建新用户** — 输入用户名，建立独立阅读空间
- **加入已有** — 输入已创建的用户名，恢复该账号的所有数据

> 同一用户名只能在一台设备在线。新设备登录旧设备会下线。

### 2. 配置 AI

右上角设置 → 选择厂商（OpenAI / Anthropic / DeepSeek / 自定义）→ 填写 API Key 和模型 → 保存。

API Key 仅存在浏览器本地，不经过第三方。

### 3. 上传小说

在书架页面拖拽 TXT/EPUB 文件，或点击"从文件夹导入"批量导入。

上传的小说自动存入服务端书库，其他人可见。

### 4. 阅读与分析

点击书架上任意小说进入阅读：
- **左侧目录** 切换章节，**底部导航** 翻章，**← → 键** 上下章
- **右上角 AI 分析** → 本章总结 / 全书分析 / AI 问答 / 笔记
- **Aa 按钮** 调节字体大小和粗细，支持暗色模式

### 5. 多设备同步

同一用户名登录后，以下数据自动同步：

| 同步 | 不同步 |
|------|--------|
| 阅读进度、AI 总结、笔记、API 配置 | 主题、字体（每台设备独立） |

切换章节或生成总结后立即推送，另一设备最多 30 秒自动更新。

### 6. 书库（局域网共享）

同一局域网下多台设备访问 `http://你的IP:5173` 可使用同一个服务端。

- **书架** — 你个人的书，有进度、总结和笔记
- **书库** — 服务端上所有人上传的所有书，点击"加入书架"即可阅读
- **从书架删除** — 只删除你的阅读数据和笔记，书仍保留在书库中

### 7. 管理后台

```bash
./admin.sh    # Linux/macOS
admin.bat     # Windows 双击
```

自动打开管理页面（首次启动生成管理员 Token），可查看/删除用户和小说。

## 技术架构

```
React 19 + TypeScript + Vite (前端)
Express + SQLite (服务端)
├─ 多 Agent 引擎: 章节总结 / 人物关系 / 剧情时间线
├─ 本地 RAG: TF-IDF + BGE Small ZH 语义检索
├─ d3-force 人物关系图谱
└─ IndexedDB (浏览器缓存) + SQLite (服务端持久化)
```

- **React 19 + Vite** — 前端框架
- **Tailwind CSS + shadcn/ui** — UI 组件
- **Express + better-sqlite3** — 后端同步 + 数据存储
- **Zustand + Dexie.js** — 状态管理 + 浏览器缓存
- **d3-force** — 人物关系力导向图谱
- **JSZip** — EPUB 解析
- **BGE Small ZH / TF-IDF** — 本地语义检索引擎
